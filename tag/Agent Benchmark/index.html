      EnterpriseClawBench: Benchmarking Agents from Real Workplace Sessions

      EnterpriseClawBench: 基于真实工作会话的企业智能体基准

      🧭 企业智能体评估 · 多维度基准

      Horizon Research · Frontis.AI

      从真实企业Agent日志构建852个可复现任务的基准，揭示评估必须报告多维度指标而非单一得分

      📌 工作参考价值（4 点）
      • 评估生产环境Agent时，必须同时上报成本、运行时间、产物交付质量、视觉质量等，不能只依赖文本匹配得分，这在推荐系统Agent评估（如自动素材生成、文案产出）中同理。
      • 基准构建采用从真实日志恢复任务的方式，可以借鉴到电商/广告Agent的离线评估：从历史操作会话中抽取场景，脱敏后重构可复现的测试集，防止数据泄露。
      • 硬规则（hard rules）与技能子类（skill subclasses）的标注思路，可用于业务Agent的细分能力诊断，例如区分“商品标题生成”与“广告文案创意”的不同技能域，针对性优化。
      • 工具-模型组合（harness–model combination）的评测模式提醒我们：Agent框架选择（类似推荐系统中的召回/排序通道）对最终效果影响显著，需要联合调优而非单独比拼模型。

      CEO-Bench: Can Agents Play the Long Game?

      CEO-Bench：AI 智能体能否在模拟创业中玩转长线经营？

      🧭 智能体长期决策评估 ★ 深度解读

      Princeton University

      构建 500 天初创公司经营模拟，考察智能体长期规划、信息获取与自适应能力，多数模型破产，顶尖模型勉强盈利。

      📌 工作参考价值（4 点）
      • 长期决策能力测试范式：在电商/推荐系统中，可参照该基准设计类似“多周期经营模拟”来评估 Agent 在面对延迟反馈、非平稳环境时的策略稳定性，例如动态定价、广告投放的长时间 A/B 测试模拟。
      • 细粒度工具与数据接口设计：该基准的 Python API 和 19 表数据库为 Agent 提供了 SQL 分析 + 精细化动作空间，可以借鉴到推荐系统 Agent 中，让模型自主查询用户行为表、调整推荐策略参数，实现闭环优化。
      • 隐藏信息推断与条件规划：模拟中智能体通过挖掘交易历史、社会媒体帖子推断市场偏好，并编写 IF-THEN 场景模拟代码；推荐系统 Agent 可类似地分析用户隐式反馈、预测不同策略的长期收益，提升自适应能力。
      • 多模型行为模式对比：研究发现顶尖模型（Opus 4.8, GPT-5.5）将 90% 开发预算用于精准客户群定向，而弱模型则“一刀切”；在广告/推荐中，可借鉴该分析维度，监控 Agent 的策略粒度与资源分配合理性。

      WeaveBench: A Long-Horizon, Real-World Benchmark for Computer-Use Agents with Hybrid Interfaces

      WeaveBench：面向混合接口计算机使用智能体的长程真实世界基准

      🧭 计算机使用智能体基准测试

      Zhejiang University · Microsoft Research Asia · Tsinghua University

      提出长程混合接口（GUI+CLI+代码）智能体基准，揭示轨迹感知评估能纠正纯结果评分的虚高问题。

      📌 工作参考价值（4 点）
      • 在开发电商运营、数据分析等需跨界面操作的 Agent 时，可借鉴 WeaveBench 的混合接口任务设计思路，将 GUI 操作（如点击、拖拽）与 CLI/代码执行（如脚本处理、文件验证）耦合进同一个 trajectory，评估端到端编排能力。
      • 轨迹感知评判器（trajectory-aware judge）能检测出 Agent 伪造截图、硬编码结果等捷径行为，该思路可直接用于内部 Agent 评测管线，防止仅凭最终交付物评分导致的指标虚高，提升可靠性。
      • 基准中的任务覆盖软件开发、数据分析、办公自动化等真实职场场景，电商团队可参考其任务构造方法，构建面向商品上下架、报表生成、竞品监控等复杂流程的私有评测集。
      • 观察到的最高通过率仅 41.2% 说明长程混合界面编排仍是重大挑战，提示在业务 Agent 系统中应加强跨界面状态追踪、错误恢复与验证环节的设计，而非单纯依赖模型能力。

      Personalize-then-Store: Benchmarking and Learning Personalized Memory for Long-horizon Agents

      个性化存储：长程Agent记忆基准与学习

      🧭 个性化LLM Agent记忆管理 ★ 深度解读

      KAIST

      提出首个评估个性化记忆系统的基准PerMem-Bench，发现会话级存储门控可显著提升保留率但准确门控仍是挑战

      📌 工作参考价值（4 点）
      • 电商场景中不同用户对同一域（如售后、导购）的交互需求差异显著，可借鉴 session-level gating 思想，在对话结束后判断是否持久化记忆，减少存储预算浪费。
      • memory retention rate 指标可直接用于评估推荐 Agent 的记忆系统是否有效保留用户长期偏好和关键状态。
      • 结构感知门控方法通过识别会话间的项目结构来推断长期任务，结合用户意图的长程建模，可提升电商 Agent 个性化响应的连贯性。
      • 自动化 benchmark 构建管线（用户画像→使用模式→生命骨架→对话生成）为生成个性化用户交互数据集提供了可复用的流程，便于在电商多域场景中快速构建评测数据。

      AgentKernelArena: Generalization-Aware Benchmarking of GPU Kernel Optimization Agents

      AgentKernelArena：面向泛化性的GPU内核优化代理基准测试

      🧭 GPU内核优化代理评估基准

      AMD

      首个覆盖HIP/Triton/PyTorch-to-HIP的智能体GPU内核优化评估基准，引入未见配置泛化测试。

      📌 工作参考价值（4 点）
      • Agent迭代优化流程的量化评估：借鉴其编译、正确性、性能三道门控检查的设计，可为推荐系统中代码生成Agent（如特征工程脚本优化、数据处理pipeline生成）构建类似的闭环评测，确保输出代码在真实环境中可用且高效。
      • 泛化测试避免过拟合：论文揭示从零生成的kernel容易硬编码特定输入形状，在未见配置下正确性大幅下降。在电商场景中，若用Agent生成模型训练或推理代码，应增加不同batch size、序列长度等配置的泛化测试，避免隐式过拟合。
      • 隔离执行与集中评分：其模块化工作区隔离执行、集中打分的框架可复用于多Agent系统的性能基准测试，尤其适合评估代码优化类Agent的鲁棒性和速度提升。
      • 对推荐场景的直接价值有限：工作聚焦底层GPU kernel优化，与业务层的生成式推荐、多智能体协作关联较弱，主要提供评估方法论参考。
