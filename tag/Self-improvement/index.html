      SEAL: Synergistic Co-Evolution of Agents and Learning Environments

      SEAL：智能体与学习环境的协同共进化

      🧭 智能体与环境协同进化训练 ★ 深度解读

      Ant Group · Westlake University · University of Michigan–Ann Arbor · University of Science and Technology of China

      利用可执行验证器为基础的失败诊断作为共享信号，协同进化训练界面与策略，显著提升低资源多轮工具使用学习

      📌 工作参考价值（4 点）
      • 训练时界面增强而不改评估基准：在工具调用训练中，可对工具描述进行轻量注解（如标注必填参数、枚举值范围），并针对失败返回结构化修复提示，不改变API或评估协议，为强化学习提供更密集的监督信号，类似推荐系统中细化反馈。
      • 失败诊断驱动的重要性加权：在GRPO等策略优化中，利用可执行规则将失败归类（参数错误、工具缺失等），根据故障可归因性分配轨迹权重，优先学习有明确修正方向的失败，可在电商多轮Agent训练中加速收敛、提升样本效率。
      • 闭环式环境-策略协同迭代：收集当前策略的失败分布（失败画像），动态调整训练界面（暴露约束提示、错误恢复建议），形成“策略暴露短板→环境针对性辅导→策略内化”的闭环，适合需要快速适应新任务或新工具的Agent系统。
      • 低资源高效起步：仅用400个训练样本便取得显著提升，适用于标注数据稀缺的业务场景（如新工具上线、冷启动），可快速验证Agent能力边界。

      Harnessing LLM Agents with Skill Programs

      将代理技能升级为可执行程序函数：HASP 框架

      🧭 Agent 技能可执行化 · Program Function ★ 深度解读

      New York University · Salesforce AI Research

      把 LLM Agent 的已知技能从被动文本变为可触发、可干预状态-动作的 Program Functions，实现推理时修正、后训练监督与技能库自进化。

      📌 工作参考价值（4 点）
      • 推理时动作硬修正：将常见失败模式（如过早终止、实体混淆）编码为 Program Function，在 Agent 循环中直接改写动作或注入纠正上下文，而不是只放入 Prompt 作为建议。这一思路可直接用于电商搜索 Agent 的多步推理，例如当检测到搜索结果过少时强制改写 Query，或在提交订单前强制检查库存。
      • 结构化干预信号用于数据筛选：每个 PF 介入记录包含时机、模式、正确性、结果四维评分，作为轨迹质量权重叠加在任务奖励上。在构建离线对话或搜索轨迹数据时，可借鉴这种多维质量打分，提升 SFT/拒绝采样数据的有效性。
      • 技能库演化需严格过滤：候选 PF 必须通过语法与模拟执行验证及教师审核才能入库，否则会严重损害性能（全过滤 60.3% vs 无过滤 36.3%）。在业务中构建可进化的策略库或规则库时，必须设计类似的自动化校验与人工审核门槛，避免技能“污染”。
      • 内部化行为纠正 PF 更有效：训练后，那些纠正模型自身行为（如搜索不充分、查询过长）的 PF 会逐渐沉默，而与输入相关的 PF 仍激活。这提示我们可以有选择地只对高频行为错误进行强制干预，并进行针对性训练来内化，从而降低上线时的判断开销。
