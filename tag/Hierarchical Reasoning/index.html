      Knowing When to Ask: Self-Gated Clarification for Hierarchical Language Agents

      知情而问：层次语言智能体的自门控澄清机制

      🧭 Agent 自门控澄清与层次推理优化 ★ 深度解读

      Amazon Web Services

      将澄清作为与导航竞争的动作，基于自身评分自门控决定何时求助，实现从强制到机会澄清的结构性转变。

      📌 工作参考价值（4 点）
      • 在层次分类/推荐任务（如商品类目预测）中，可将“请求用户补充属性”视为与预测动作同尺度评分的选项，用阈值决定是否触发交互，从而在关键中间节点精准获取信息。
      • 分离帮助定位（where）与答案质量（what），通过受控答案质量实验诊断系统瓶颈，若帮助定位稳定而答案质量差导致增益消失，则瓶颈在答案源而非提问策略。
      • ISE（信息寻求有效性，即帮助后下一步导航正确的比例）可作为局部效用诊断指标，比最终准确率更早发现提问策略的质量变化，避免仅看最终结果。
      • 在智能体决策链中引入“动作评分+自门控”范式，无需外部不确定性估计器，即可让信息寻求行为结构化可观测，便于调试和优化。

      Chain-of-Procedure: Hierarchical Visual-Language Reasoning for Procedural QA

      面向视觉过程问答的分层推理框架与基准

      🧭 视觉过程问答 · 多模态分层推理

      提出ProcedureVQA基准和CoP层次推理框架，通过先检索后细化的方式增强VLM的过程推理能力

      📌 工作参考价值（4 点）
      • Agent 多步决策借鉴：CoP 的“先粗粒度检索 → 细粒度步骤分解”架构可直接用于电商场景的过程式 Agent，如根据订单状态图片自动检索售后退款流程并定位下一步操作。
      • 多模态 RAG 实现：利用视觉线索检索结构化过程知识库的方法，能迁移到商品手册、安装教程的智能问答系统，通过图像匹配相关文档段落再生成答案。
      • 数据构建思路：ProcedureVQA 的构建方式可用于内部评测 Agent 的多模态过程推理能力，暴露模型在细粒度跨模态对齐上的短板。
      • 流程优化 trick：CoP 的语义分解模块对于长文本步骤的细化，可用于将用户粗糙的意图（如“退货”）分解为可执行的子步骤，提升推荐或客服 Agent 的指令可操作性。
