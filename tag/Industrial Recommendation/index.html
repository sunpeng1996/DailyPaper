      Gryphon: A Unified Architecture for Semantic-ID Generation and Item-Level Scoring in Industrial Recommendations

      Gryphon：统一语义ID生成与物品级评分的生成式推荐架构

      🧭 生成式推荐 · Semantic ID · item-level scoring ★ 深度解读

      Yandex

      在生成式检索中引入联合训练的物品级评分模块，解耦beam likelihood与最终排序，解决SID碰撞和得分失校准

      📌 工作参考价值（4 点）
      • 用beam search做ID候选生成，再加一个轻量级评分器重排序：工程上可复用的架构模式，beam likelihood仅用于控制候选池入口，最终排序由Item-Level Scoring Module（ILSM）接管，避免beam失校准影响回召；尤其适合SID碰撞严重的动态目录场景。
      • ILSM复用编码器用户状态，成本极低：只需一层用户-物品交叉注意力+MLP，几乎不增加推理耗时，可直接集成到现有生成式检索模型中。
      • 联合训练目标：生成损失 + 带流行度纠正的next-item prediction损失：共享编码器，使同一用户表示同时支持SID质量与物品级相关性，训练稳定，可推广到多目标优化。
      • 在线A/B证明单一生成式模型可替代复杂多路召回：Gryphon取代了15+候选源与预排序，候选量从3000降至1000，核心指标持平，大幅降低系统复杂度和运维成本，对电商推荐系统多路融合极有参考价值。

      LoopFM: Learning frOm HistOrical RePresentations of Foundation Model for Recommendation

      LoopFM：基于基础模型历史表征的推荐知识迁移

      🧭 知识蒸馏优化 · 历史表示结构化 ★ 深度解读

      Meta AI

      将大模型历史中间层嵌入压缩并序列化为小模型输入特征，打破标量知识蒸馏的带宽瓶颈。

      📌 工作参考价值（4 点）
      • 离线生成用户侧序列特征：将大模型已产生的历史交互 embedding 按键（如 user ID）压缩成时间序列，作为 lightweight 模型的直接输入，无需实时推理大模型，显著降低在线延迟。
      • 正交填充知识缺口：LoopFM 序列提供跨域、跨特征交互信息，与标量 KD 互补；在线上可并行部署两条迁移通道，提升整体迁移率（内部实验迁移率翻倍）。
      • 以压缩控制成本，以结构保留时序信号：用自动编码器 + Matryoshka 维度灵活 + INT4 量化，可实现 4× 存储压缩；序列化方式比简单池化更好（DMIN attention 优于 mean-pool），但 sum-pool 也能捕获大部分增益，可按成本选择。
      • 容量差距越大收益越高：小模型直接学高基数 ID 序列效果差，而 LoopFM 提供的低维、信息浓缩表示使小模型直接受益，这适合电商推荐中的重模型-轻模型架构。

      Rec-Distill: An Industrial Distillation Pipeline for Large-Scale Recommendation Models

      Rec-Distill：面向工业推荐的大规模模型蒸馏框架

      🧭 推荐系统 · 知识蒸馏 · 大规模模型压缩 ★ 深度解读

      ByteDance AML · ByteDance

      解耦式蒸馏框架将大模型（24B参数、20K序列）知识高效迁移至轻量线上模型，迁移率超60%并取得商业收益

      📌 工作参考价值（4 点）
      • 解耦式蒸馏推进大模型复用：教师与学生对训练完全解耦，教师持续流式更新并缓存logits，学生异步消费。可参考此“1-to-N”范式，一个大教师同时服务多个小模型（如召回、粗排、精排），显著降低训练成本，实现线上模型不限制实验规模。
      • 黑盒蒸馏+loss缩放：使用教师输出logits的交叉熵损失，简单有效，且观察到蒸馏损失尺度比主任务损失小两个数量级时效果差，需通过调大系数α对齐量级。实际部署时可直接采用CE蒸馏，并手动调大α。
      • 去偏机制解决样本分布不匹配：教师和学生训练数据采样策略不同导致logits偏差，通过将教师logits用学生侧去偏公式重校正（𝑇2′=𝑓𝑆(𝑇1)），再与学生去偏logits蒸馏，可提点0.1%。电商场景普遍存在采样偏差，该方法可复用。
      • 混合批-流式管道保证实时性：批训练阶段用大量数据快速收敛，流式阶段从高速存储读取教师最新logits追加到学生流数据，实现分钟级延迟适应数据分布变化，避免线上掉点。适合实时推荐系统的高频更新需求。

      SIREN: Unified Multi-Granularity Semantic Interaction for Multi-Modal Lifelong User Interest Modeling

      SIREN：多粒度语义交互的统一多模态终身兴趣建模

      🧭 多模态终身兴趣建模 · 语义ID ★ 深度解读

      Tencent Inc. · Xiamen University

      在工业推荐的两阶段框架中，通过语义ID和相似度桶实现多模态信号的早期融合，兼顾效果与部署效率。

      📌 工作参考价值（4 点）
      • 语义ID生成与使用：利用RQ-VAE将多模态embedding离散化为分层语义ID，并通过前缀编码保留层级语义，可作为item的稳定通用token，既用于GSU的硬检索（倒排索引，降低90%以上在线成本），又作为ESU中细粒度特征，可迁移至电商商品的生成式推荐或冷启表征。
      • 早期融合代替晚期融合：将相似度桶和语义ID直接拼入item表征，与ID特征一起进入target-conditioned Transformer进行统一序列建模，而不是在序列聚合后再融合。这让多模态信号参与注意力权重和兴趣表征的联合学习，可应用于短视频、广告等场景的多模态长期行为建模。
      • 相似度桶化与语义ID的互补设计：相似度桶提供粗粒度的目标相关度，语义ID捕获细粒度的物品语义和协同异质性。分析表明二者互补，且单纯增加桶数无法弥补细粒度缺失。在电商场景中可借鉴该组合特征方式，提升老客与新品的匹配精度。
      • 冷启动增益：多模态信号显著提升低活用户和新广告的效果（GMV增益可达整体平均的1.4~3.6倍），对于电商新品冷启、小语种用户等场景有直接参考价值。
