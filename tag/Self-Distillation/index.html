      HINT-SD: Targeted Hindsight Self-Distillation for Long-Horizon Agents

      HINT-SD：面向长程智能体的目标化后见自蒸馏

      🧭 长程 Agent 后训练 · 目标化后见蒸馏 ★ 深度解读

      KAIST · DeepAuto.ai

      通过选择失败相关动作进行自蒸馏，在不损失效率的前提下为长程 LLM Agent 提供精准过程监督

      📌 工作参考价值（4 点）
      • 选择性过程监督：长程 Agent RL 训练中不要均匀地对所有步骤蒸馏反馈。先用策略自身对失败轨迹进行后见分析（Hindsight Analyzer），仅挑选出导致失败的关键步骤，再对这些步骤进行带反馈的自蒸馏，避免大量正确或无关步骤的噪声更新。在电商购物助手、多轮工具调用等场景中，可直接复刻“复盘→定位错误→精准纠正”的范式。
      • 反馈放置的时机很关键：实验表明，将纠正反馈插入到发生错误的动作之前（而非整个轨迹的开头），教师与学生的上下文分布更匹配，纠正效果明显更好。在实施反馈条件蒸馏时，应将反馈准确地放置在相关步骤的上下文中，避免因前缀不同导致分布错位。
      • 自蒸馏作为本地化的教师：无需外部大模型，训练中的 EMA 策略本身即可充当反馈生成器，节省成本。这在资源有限的场景（如在自有模型上持续训练 Agent）中尤为实用，可作为一个轻量、自包含的在线蒸馏组件集成进训练流程。
      • 效率启发：选择性蒸馏大幅缩短训练步时和显存消耗（2.26× 步时降低，1.48× 显存降低）。对于需要批量处理大量长对话的推荐 / Agent 系统，采用“先判断哪些步骤需要学习”的策略可显著提升迭代效率和资源利用率。

      SD-Search: On-Policy Hindsight Self-Distillation for Search-Augmented Reasoning

      SD-Search：策略事后自蒸馏实现搜索增强推理的步骤级监督

      🧭 搜索增强推理 · 事后自蒸馏 ★ 深度解读

      Kuaishou Technology

      提出事后自蒸馏框架，从策略自身获取搜索步骤的细粒度监督，无需外部教师或额外标注

      📌 工作参考价值（4 点）
      • 从策略自身挖掘步骤级信号：同一模型在训练时分别以“学生”（仅推理时上下文）和“教师”（额外看见同组滚动的事后摘要）两种视角运行，用分布匹配提取各搜索决策的细粒度反馈，无需外部教师或昂贵标注。可在推荐 Agent 的多步交互中复用，对关键决策步骤（如召回、重排）施加 token 级自蒸馏。
      • 事后信息块的精巧设计：通过屏蔽答案和思考内容、保留查询与二元结果标签（CORRECT/INCORRECT）构建对比，迫使教师隐含地评价哪些查询值得执行。在对话或推荐 Agent 中，可用历史动作和成功/失败标签构建类似的事后上下文，强化正向动作模式。

      • 无额外推理开销的 RL 融合：自蒸馏仅增加一次辅助前向和损失项，不修改奖励或优势计算，训练耗时仅增 15.5%，推理完全不变。低成本地提升了多跳问答的查询质量，而不增加搜索次数，对线上 Agent 服务的效率优化有启发。
