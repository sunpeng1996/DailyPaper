      University of Alberta

      📌 工作参考价值（4 点）
      • 推荐系统上线时常有离线日志数据，LinOtO 的「安全基线 + 预算探索」框架可直接用于冷启动阶段：先用离线保守策略（LCB）累积足够预算，再逐步过渡到在线探索，避免早期高遗憾。

      Amazon AGI

      提出CAS框架，通过动态混合训练及GPU/存储交换，实现数百个per-document cartridge的联合训练，消除多缓存混合的性能塌陷，结合检索较文本RAG节省3-4倍token。

      📌 工作参考价值（4 点）
      • 可组合的KV缓存训练范式：在需要为大量静态文档（商品政策、FAQ、医典）构建可重用缓存的场景，直接采纳混合可见性训练（P<sub>iso</sub>=0.75，每样例以一定概率加入其他文档的cartridge作为干扰），可完全避免多缓存拼接时的性能暴跌（从26%回至78%），能直接复用到电商长文档问答的缓存构建中。

      • 混合RAG与压缩缓存的推理策略：Cartridge RAG用检索选出相关文档的压缩缓存代替原始文本块，在LongHealth上以3.7倍更少prompt tokens达到同等精度。对于电商搜索结果总结或客服知识库问答，可大幅减少输入长度、加速生成，同时保持片段级检索的准确性。在信息高度密集的表格型内容上需谨慎控制激活的cartridge数量，否则性能下降，这一点对金融/库存类文档压缩有直接启示。

      Cascading Hallucination in Agentic RAG: The CHARM Framework for Detection and Mitigation

      代理RAG中的级联幻觉检测与缓解：CHARM框架

      🧭 代理RAG · 级联幻觉检测与缓解 ★ 深度解读

      University of North Dakota

      提出CHARM框架，通过跨阶段事实验证、一致性跟踪和置信度监控，实现89.4%的级联幻觉检测率和82.1%的错误传播减少。

      📌 工作参考价值（4 点）
      • 多步推理流水线的被动检测层：电商推荐Agent常涉及查询解析→召回→排序→生成解释等多步推理，可借鉴CHARM在不替换原有架构的前提下，引入跨阶段事实验证（SFV）和语义漂移跟踪（CSCT），捕获中间步骤的事实错误与逻辑偏移，避免错误积累导致最终推荐结果看似合理实则错误。
      • 高价值场景的双模验证：对于大促选品、合规文案等高风险决策，可采用平行验证代理（PVA）进行独立复核，以2倍计算成本换取95.2%的缓解成功率，适合金融、医疗等强监管领域。
      • 置信度传播监控：在生成式推荐或对话流中，可借鉴置信度监视器（CPM）的思想，用贝叶斯跟踪模型自信度的变化，防止低置信度的中间结论被后续步骤当作高置信信号使用，尤其在解释生成环节防止自欺欺人的“自信幻觉”。
      • 可配置阈值与业务风险对齐：CHARM各组件阈值可按业务域校准，例如搜索Query改写阶段可适度降低NLI阈值以提升召回，而合规内容生成则可收紧阈值，并启用流水线回滚（PRR）确保最终输出基于正确上下文。

      📌 工作参考价值（4 点）
      • 可直接复用单次前向传播提取置信度的思路：从 LLM 的 hidden state 计算语义差距、注意力熵、输出概率差，无需多轮采样，推理零额外开销，适合在线重排场景。

      Distributional Approximate Nearest Neighbour Search for Uncertainty-Aware Retrieval

      🧭 推荐系统 · 向量检索与探索 ★ 深度解读

      aampe · Antwerp, Belgium

      在 ANN 索引中为每个物品采样多个嵌入副本，用不确定性隐式驱动检索阶段的探索，大幅提升长尾覆盖

      📌 工作参考价值（4 点）
      • 直接在 Faiss 等现有 ANN 索引上实现：每个物品预先采样多个嵌入加入索引，查询时也采样用户嵌入并检索去重，无需修改模型或索引后端，即可将嵌入不确定性转换为长尾物品的曝光机会。
      • 自适应采样预算：对冷门、高不确定性的物品分配更多样本（如 S=2~10），头部物品保持单样本，可有效控制索引膨胀，同时将探索算力集中在需要曝光的尾部。
      • 用交互计数简单构造“合成后验”：令 σ_i ∝ 1/(1+n_i)^γ，以此从点估计中生成模拟贝叶斯不确定性的噪声样本，低成本实现类似汤普森采样的检索阶段探索。

      ForeSci: Evaluating LLM Agents for Forward-Looking AI Research Judgment

      ForeSci：评估LLM智能体前瞻性AI研究判断的时序基准

      🧭 Agent 前瞻性判断评估基准

      Southeast University · Beijing Zhongguancun Academy · Duke Kunshan University

      构建时间屏蔽的基准，揭示LLM代理在做前瞻研究决策时证据组织提升可追溯性但存在证据-决策脱节问题

      📌 工作参考价值（4 点）
      • 时间屏蔽的离线知识库构建：在电商推荐场景中，可借鉴其按时间切割数据的方式，构建严格避免未来信息泄露的离线评估集，评估模型对用户未来兴趣的预测能力。
      • 显式证据组织提升可解释性：论文表明让Agent显式组织证据（如引用历史论文片段）能增强事实支撑，可迁移到电商Agent的决策链设计，例如要求推荐Agent在输出商品列表前必须引用用户历史行为证据，提升可追溯性。

      📌 工作参考价值（4 点）
      • 代理实验（Proxy Experiments）的外推风险：在推荐/Agent 研发中，若用小规模模型或代理任务（如点击率预测代在离线收益）来筛选方案，必须验证结论能否跨规模泛化，否则可能错失真正有效的大模型配置。可借鉴论文中对外部效度和构念效度的警示，设计从小到大的校准实验。
