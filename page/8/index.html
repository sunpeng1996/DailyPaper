      超越全局重规划：跨设备智能体的分层恢复

      Shanghai Jiao Tong University · Shanghai Innovation Institute · Southeast University · Tsinghua University

      提出 H-RePlan 分层重规划框架，分离设备局部策略恢复与全局重规划，结合 HeraBench 故障注入基准，大幅提升多设备 Agent 任务完成鲁棒性并降低 token 消耗。

      📌 工作参考价值（4 点）
      • 分层故障隔离与恢复：在电商搜索/推荐系统的多服务 Agent 编排中，可借鉴设备本地策略恢复与全局重规划分离的思路，区分 API 故障、参数错误等局部可修复错误与需要重新规划跨服务流程的全局错误，避免代价高昂的全量重试。

      • 故障注入测试基准：参考 HeraBench 构建方法，为推荐/搜索 Agent 工作流注入策略级（如服务超时、数据格式错误）和设备级（如节点宕机）故障，系统评估 Agent 在动态异常下的成功率与指令遵循度，指导鲁棒性优化。
      • 紧凑跨层抽象通信：在全局编排器与局部执行器之间使用结构化故障信号（如错误码、策略空间状态），减少 LLM 对话轮次和 token 消耗，可用于多 Agent 协作的轻量级信息传递设计。

      Contagion Networks: Evaluator Bias Propagation in Multi-Agent LLM Systems

      传染网络：多智能体LLM评估偏差传播的量化与抑制

      🧭 多智能体LLM评估偏差传播与抑制 ★ 深度解读

      Qilu Institute of Technology, School of Software Engineering

      提出传染网络框架，量化多智能体LLM中评估偏差传播，揭示同模型抑制与委员会多样性降低传染效应

      📌 工作参考价值（4 点）
      • 同模型评估者池优先：在电商搜索/推荐的多智能体架构（如 query 评估、对话推荐）中，尽量使用同一模型家族但不同角色提示的智能体相互评估，避免跨模型（如 GPT-4o 评估 DeepSeek）引入的偏差放大（同模型传染系数仅 0.14–0.30，跨模型可达 0.85–1.3）。
      • 策略熵作为健康监控指标：实时跟踪各智能体的策略权重熵 H(w)，若持续下降逼近低值，提示可能存在传染偏向单一策略，需触发多样性注入或评估者轮换。
      • 评估者委员会降传染：对关键决策节点，采用 k≥3 个评估者组成委员会，可将有效传染降低 72% 以上，是一种低成本防御机制，适合工业级多智能体系统。
      • 部署前测量 Γ 矩阵：新建多智能体系统时，先跑一轮小规模传染矩阵测量，若谱半径 ρ(Γ)>1（全连接拓扑）或存在单跳 γ>1（链拓扑），需调整网络拓扑或评估配置，避免级联坍塌。
