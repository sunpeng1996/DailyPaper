      Your Agents Are Aging Too: Agent Lifespan Engineering for Deployed Systems

      AI Agent也会老化：部署系统中的Agent寿命工程

      🧭 Agent系统寿命工程与老化诊断 ★ 深度解读

      The University of Texas at Austin

      首次定义Agent老化并构建纵向可靠性基准AgingBench，通过四种机制和诊断探针实现组件级归因。

      📌 工作参考价值（4 点）
      • 部署后的纵向监控：Agent可靠性会随时间退化，仅靠单次基准不足。电商推荐Agent（如长期对话助手）应建立老化曲线和半衰期指标，监控事实精度衰减。
      • 写时值保留策略：压缩老化表明摘要会丢弃关键细节（如金额、约束）。在记忆写入时，可注入明确保留指令，或用结构化字段保存数值，防止“每日服药”这类泛化丢失。
      • 干扰诊断与检索增强：干扰老化源于相似记忆混淆，可借鉴counterfactual探针区分是写入遗漏还是检索失败。对应到多用户推荐中，需对相似实体的记忆做硬性区分（如user ID后缀），并诊断是向量检索不准还是记忆未写入。
      • 维护后回归检测：memory recompaction等操作会引发性能断崖。Agent上线后应在每次维护操作后运行关键事实的回归探测，对突变指标自动告警，类似软件CI/CD流程。

      DVAO: Dynamic Variance-adaptive Advantage Optimization for Multi-reward Reinforcement Learning

      DV AO：面向多奖励强化学习的动态方差自适应优势优化

      🧭 多奖励强化学习 · 优势组合优化 ★ 深度解读

      Alibaba Cloud Computing

      动态按组内奖励方差自适应调整多目标优势权重，解决 GRPO 多奖励标量化的幅度爆炸与目标孤立问题

      📌 工作参考价值（4 点）
      • 多目标奖励的可学习聚合：在电商推荐/Agent 多目标优化（如准确率+多样性+延迟）中，现有做法往往直接固定权重标量化奖励或优势，导致某一目标梯度主导。DV AO 提供了完全数据驱动的自适应加权方案，无需人工调参，直接基于每个目标的组级方差动态平衡信号强度，可应用于生成式推荐的 multi-reward 对齐。
      • 稳定训练的技巧：DV AO 从理论上保证了优势幅度低于直接奖励组合，避免策略梯度爆炸。在 RL 微调推荐模型或 Agent 策略时，可直接用于处理准确度、格式合规、长度等冲突目标，防止训练崩溃。
      • 交叉目标正则化机制：通过优势计算对单一目标梯度的重新缩放，隐式地促进目标间协同优化，避免某个简单目标淹没困难目标。这对 Agent 多轮任务中同时优化任务成功率与工具调用格式很有价值。
      • 轻量实现：DV AO 不引入额外可学习参数，完全基于组内方差的后计算，易于集成到现有的 GRPO 或 PPO 训练流程中，适合快速实验。

      Selective Test-Time Compute Scaling for Click-Through Rate Prediction via Uncertainty-Triggered Feature Path Exploration

      面向CTR预测的选择性测试时计算缩放：不确定性触发特征路径探索

      🧭 测试时优化 · CTR预测 · 不确定性估计 ★ 深度解读

      Alibaba Group

      提出训练无关、模型无关的测试时框架UTTSI，根据样本不确定性自适应分配多路径探索，显著提升CTR预测性能

      📌 工作参考价值（4 点）
      • 不确定性感知的推理预算分配：在电商推荐中，用户/物品交互稀疏导致部分样本预测不可靠。UTTSI 通过 Count-Min Sketch 统计特征频率作为先验，结合模型 logit 置信度（或梯度属性）计算样本不确定性，从而为高不确定性样本投入额外计算，低不确定性样本快速通过。该方法无需修改模型，可轻量植入现有线上服务，尤其适合特征稀疏的业务。
      • 多路径特征探索与一致性加权：对高不确定性样本，基于频率和属性分数随机采样生成多个特征子集，并行做多次前向预测，再用一致性加权聚合，可平均化噪声、提升鲁棒性。工程上可将多个子路径预测部署在并行 worker 上，保持单次前向延迟不变，适合有GPU平算资源的在线推理。
      • 自适应特征过滤：通过离线计算每个特征场的可靠性阈值（分位数），推理时过滤低可靠性特征（频率+属性双低），简单有效。可单独作为一种特征去噪 trick 应用于生成式推荐或 Agent 的输入预处理，在保证高置信度特征的同时抑制噪声。
      • 测试时计算缩放的范式迁移：本工作表明，即使不改变模型结构和训练流程，仅在推理时按需分配计算，也能带来可观的线上收益（A/B CTR 相对提升 5.3%）。对于有严格延迟要求但有额外算力的电商推荐系统，可以借鉴这种“推理时计算预算分配”思想，对低置信度请求进行多路召回或额外特征融合，而不影响基础链路。

      Self-Balancing Gradient Allocation for Heterogeneity-Aware Feature Generation in Click-Through Rate Prediction

      面向异质性特征生成的自平衡梯度分配，用于CTR预测

      🧭 生成式推荐 · 异质性特征生成 · 难度感知 ★ 深度解读

      Alibaba Group

      提出统一可学习难度信号，同时驱动自平衡损失和难度引导注意力，解决生成式CTR中不同字段重建难度不均衡问题。

      📌 工作参考价值（4 点）
      • 自平衡损失加权：将每个特征字段视为独立任务，借鉴多任务同方差不确定性加权，损失设为 exp(-s)*loss + s/2，无需手工调权，自动将更多梯度分配给高基数ID、序列等难重建字段，冷启动受益明显。
      • 难度引导注意力调制：在HSTU去噪网络中，用 exp(-s/2) 缩放查询向量，抑制收敛后字段的注意力，增强困难字段的信息聚合，与损失加权使用同一 s 信号，保持一致性。
      • 统一难度信号：仅需学习每个字段一个标量 s_i，零额外超参数，与现有生成式预训练框架（如 DGenCTR）无缝集成，离线预训练开销极低。
      • 冷启动与长尾增益：该方法对稀疏用户和长尾物品提升更大，生产环境中可针对这些分段部署，获得显著业务指标提升（在线 A/B 冷启动用户 CTR +9.2%）。
