      Harnessing LLM Agents with Skill Programs

      将代理技能升级为可执行程序函数：HASP 框架

      🧭 Agent 技能可执行化 · Program Function ★ 深度解读

      New York University · Salesforce AI Research

      把 LLM Agent 的已知技能从被动文本变为可触发、可干预状态-动作的 Program Functions，实现推理时修正、后训练监督与技能库自进化。

      📌 工作参考价值（4 点）
      • 推理时动作硬修正：将常见失败模式（如过早终止、实体混淆）编码为 Program Function，在 Agent 循环中直接改写动作或注入纠正上下文，而不是只放入 Prompt 作为建议。这一思路可直接用于电商搜索 Agent 的多步推理，例如当检测到搜索结果过少时强制改写 Query，或在提交订单前强制检查库存。
      • 结构化干预信号用于数据筛选：每个 PF 介入记录包含时机、模式、正确性、结果四维评分，作为轨迹质量权重叠加在任务奖励上。在构建离线对话或搜索轨迹数据时，可借鉴这种多维质量打分，提升 SFT/拒绝采样数据的有效性。
      • 技能库演化需严格过滤：候选 PF 必须通过语法与模拟执行验证及教师审核才能入库，否则会严重损害性能（全过滤 60.3% vs 无过滤 36.3%）。在业务中构建可进化的策略库或规则库时，必须设计类似的自动化校验与人工审核门槛，避免技能“污染”。
      • 内部化行为纠正 PF 更有效：训练后，那些纠正模型自身行为（如搜索不充分、查询过长）的 PF 会逐渐沉默，而与输入相关的 PF 仍激活。这提示我们可以有选择地只对高频行为错误进行强制干预，并进行针对性训练来内化，从而降低上线时的判断开销。

      SNLP: Layer-Parallel Inference via Structured Newton Corrections

      结构化牛顿层并行：用廉价代理雅可比实现Transformer层间并行推理

      🧭 层并行推理 · 结构化牛顿校正 ★ 深度解读

      Red Hat AI Innovation · MIT-IBM Watson AI Lab · Core AI, IBM

      将层间隐藏状态视为非线性残差方程，用结构化代理雅可比替换精确雅可比，结合训练正则化，实现层并行加速。

      📌 工作参考价值（4 点）
      • IDN 正则化让残差块的雅可比逼近 I，可移植到生成式推荐的 Transformer 编码器/解码器训练中，降低层间依赖，为后续并行化创造空间。
      • 推理时使用 IDN（即 hl+1 = f_l(hl-1) + (h_l - h_l-1)）做轻量校正，可实现层间并行评估，结合算子融合与 chunk 划分，在0.5B模型上获得2.3×加速，此模式可试用于电商搜索/推荐模型的实时推理加速，特别是当模型深度较大时。
      • HCN 利用架构内残差混合矩阵（如mHC）作为代理雅可比，避免任何雅可比估计开销；若系统已有流式记忆或路由结构，可直接复用其权重作为并行校正算子。
      • SNLP 训练正则（让有限步牛顿迭代解逼近序列解）同时改善序列PPL（降低4.7%~23.4%），可作为通用的训练正则项提升模型质量，即使不启用并行推理也值得尝试。

      LongMINT: Evaluating Memory under Multi-Target Interference in Long-Horizon Agent Systems

      MINTEVAL：在长时程多目标干扰下评估 Agent 记忆系统

      🧭 Agent 记忆评估与干扰鲁棒性 ★ 深度解读

      UNC Chapel Hill · The University of Texas at Austin

      一个面向干扰密集、多领域、长时程记忆的基准，揭示现有记忆系统在回溯与聚合推理上的严重不足

      📌 工作参考价值（4 点）
      • 在电商/推荐场景中，用户偏好频繁更新且常需回溯历史状态（如曾经喜欢某品牌），现有记忆系统由于过度依赖追加新记忆、缺乏细粒度修改/删除，导致早期信息被覆盖。建议：引入显式时间戳或版本标记，并设计支持 CRUD 的原子记忆操作，以减少干扰。
