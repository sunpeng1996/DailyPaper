      🧭 知识注入 · 混合自蒸馏 · 分布对齐 ★ 深度解读

      Carnegie Mellon University · University of Toronto · University of Illinois Urbana-Champaign · Princeton University · Cornell University

      提出MIXSD，通过混合基础模型自身专家与朴素条件采样构建分布对齐监督，大幅减轻知识注入时的灾难性遗忘。

      📌 工作参考价值（4 点）
      • 用模型自身的语境采样作为监督信号：在注入新知识时，不直接使用人工书写的目标文本，而是用基础模型在“看到事实”与“未看到事实”两种条件分布下采样的token混合生成训练目标，使监督更贴近模型原有分布，减少分布偏移带来的遗忘。

      • 用 Fisher 对齐比率诊断遗忘风险：发现参数更新方向（而非幅度）与遗忘强相关，可在微调过程中监控该指标，提前预警能力退化；对于 Agent 多步推理能力保持尤其重要。

      服务器端 HTTP 日志自动重构读集，实现多智能体共享 NL 状态的乐观并发控制，零 agent 改动。

      📌 工作参考价值（4 点）
      • 对于多 agent 协作修改共享 NL 状态（如商品描述、推荐策略片段），可借鉴 DeliveryLog 自动追踪 agent HTTP GET 请求的版本，在提交时做跨键检查，无需 agent 侧编码读集追踪，大幅降低开发负担。
      • 采用专用 shard 拓扑（每个 agent 拥有独立的写键，只读取共享参考 shard）可天然避免写冲突，同时利用乐观锁防止跨 shard 陈旧读取。电商场景中，多个 agent 可并行修改不同商品的详情，共享品牌规范等只读信息。

      • 该中间件对现有 agent 框架透明，可直接部署为旁路服务；在 HTTP/1.1 场景下几乎零改造，HTTP/2 下可通过 ARSI 模式强制完备读集重建。
