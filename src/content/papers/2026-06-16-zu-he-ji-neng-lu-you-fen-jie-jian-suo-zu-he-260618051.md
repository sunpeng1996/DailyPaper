**动机**  
LLM Agent 的工具库日益庞大（如 MCP 生态已达数千技能），但真实任务通常需要*组合*多个技能，而非单选。现有研究多聚焦单技能路由，面对“下载数据集→转换格式→生成报表”这类多步请求无能为力。为此，论文提出组合技能路由问题，并在数千真实技能上构建基准，量化瓶颈所在。

**方法关键点**  
- **三阶段框架 SKILLWEAVER**：分解（LLM 将复杂查询拆为原子子任务）→ 检索（bi-encoder + FAISS 为每个子任务召回 Top-10 候选技能）→ 组合（基于兼容性 DAG 规划最终技能链）。  
- **技能感知分解 SAD**：核心创新。将 Pass-1 检索结果的 Top-H 技能作为 hint 注入分解的 LLM Prompt，形成检索→分解的反馈环。仅一次迭代便使分解准确率（DA）从 51.0% 升至 67.7%。  
- **组合规划**：在候选技能上利用 I/O 类型、类别重合度等计算兼容性，选出兼顾相关性与步骤连贯性的技能序列。  
- **基准 COMPSKILLBENCH**：从 MCP 生态抽取 2209 个真实技能、24 个类别，生成 300 条组合查询（2~5 步），带有 skill ID 和类别 ground truth。

**关键实验与数字**  
- **瓶颈诊断**：标准 LLM 分解仅达 34.2% CatR@1（步骤级类别召回），且 DA=51.0%；当分解步骤数正确时，CatR@1 升至 41.2%，证实分解粒度是第一瓶颈。  
- **SAD 效果**：单次 SAD 使 DA 提升 32.7%（+16.7pp），DA±1 达 84.3%；在人类风格查询上（零文本重叠）DA±1 从 30.5% 提升至 50.5%（+66% 相对）。  
- **泛化性**：类别留出实验中 SAD 仍带来 +35.6% 的相对 DA 提升，技能级留出 +23.2%，表明 SAD 学到的是技能库的*结构词汇*而非记忆特定映射。  
- **上下文压缩**：SKILLWEAVER 将暴露给 LLM 的工具 token 从 88.4 万降至约 1160（99.9% 减少），且端到端链式执行完成率 76.7%。  

**一句话总结**  
*组合技能路由的瓶颈在分解粒度，而不是检索容量；用检索结果反哺分解的 SAD 循环，能以极小成本大幅提升复杂任务的技能组合质量。*
