**动机**：时间序列预测在统计模型层面已日趋成熟，但在实际业务决策中，一个统计上合理的基线预测往往需要结合节假日、促销计划、外部事件等上下文信息进行二次修正才能落地。这种“最后一公里”的修订过程过去主要依赖人工经验，缺乏结构化和可审计性。本文将该问题形式化为 **last‑mile forecasting**，即从基线预测到业务可用预测的约束化修订，并提出一种 LLM‑agent 框架来解决。

**方法关键点**
- **系统架构**：在预测主干（如 TimesFM）之上构建 Agent，将数值外推和上下文修正彻底解耦，主干负责基线，Agent 负责可追溯的修订。
- **统一工作区**：保持一个包含历史值、不可变基线、可编辑修订值的数据表，所有修改只能通过受限动作（范围缩放、点位覆写、修订列表合并）作用于 y_final，确保修改变更完整记录。
- **基于工具的证据获取**：Agent 通过日历查询、历史检索、记忆查询等工具收集证据，并将证据转化为结构化修订提案（含方向、幅度、置信度），再驱动修订动作。
- **长时预测的 map‑reduce 分解**：将长预测区间按事件窗口分解，每个窗口由本地推理器独立生成修订信号后再汇总执行，保持过程清晰可审计。
- **记忆与自改进**：在真实值出现后，对比基线、修正值和实际值，生成反思记录存入记忆库；后续预测时可检索这些记忆，以校准修正策略，实现跨 session 的自提升。

**实验数据与结果**
基于携程某国内航线的每日机票销量数据（2024‑01‑01 至 2026‑05‑05），设计三组 case study：
- **节假日修正**：对 2026 年春节窗口，相比 Prophet 和 TimesFM，MAE 分别降低 80.0% 和 88.2%，MAPE 从 155.95%／262.76% 降至 32.84%。
- **长时多事件预测**：126 天区间含四个节假日，整体 MAPE 降至 18.91%，每个事件窗口误差均为最低，展示了按事件精准修正而非全局统一偏移的能力。
- **自改进机制**：在连续三周的非事件窗口上，带记忆的 Agent 在第三周 MAE（60.10）显著低于无记忆对照组（92.94），验证了反思记忆的有效性。

**关键结论**：将 LLM Agent 的角色从预测生成转向预测的上下文修正，通过约束化、痕迹化的工作流，能显著提升预测的业务可用性与审计性，为“预测最后一公里”提供系统化方案。
