**动机**  
现有 Agent 系统将经验总结为文本技能注入 Prompt，但文本技能是被动建议，未规定何时触发、如何修正动作，实际执行中常被模型忽略。为了弥合“经验表述”与“可靠行为控制”之间的差距，需要一种能将技能转化为可执行干预的机制。

**方法关键点**  
- **Program Function (PF)**：每个技能被封装为一个 PF，包含 `should_activate`（判断是否触发）和 `intervene`（执行动作改写或上下文注入），直接插入 Agent 推理循环。  
- **推理时 Harness**：基础策略提议动作后，检索相关 PF，评估激活条件，若激活则修改动作或注入纠正性上下文，无关时原动作通过。  
- **结构化监督信号**：每次 PF 介入记录**时机、模式、正确性、结果**四维得分，组合成轨迹质量指标，用于后训练数据筛选。  
- **后训练内化**：支持 SFT（直接学习矫正动作）、拒绝采样（按任务成功与 PF 得分选择轨迹）、在线蒸馏（用教师轨迹纠正学生行为）。  
- **技能库自进化**：从剩余失败中总结候选 PF，经过**可执行验证**与**教师审核**双重过滤后入库，防止技能污染。

**关键实验结果**  
– 在 Qwen2.5-7B-Instruct 上，仅推理时 PF 介入（无训练）在 Web 搜索推理平均准确率从 ReAct 的 31.2% 升至 51.0%（提升 25 个百分点），加教师选择升至 56.2%。  
– 固定技能库下，PF 引导的拒绝采样达 59.3%，在线蒸馏达 62.5%；引入进化后，PF-Evolve + RS 达 60.3%，数学推理从 38.8% 提至 45.4%，编码平均 pass@1 从 63.4% 提至 69.9%。  
– 消融表明，去除时机、模式、正确性、结果任一信号平均下降 12~15.5 个百分点；无过滤技能进化致使性能暴跌至 36.3%。  
– PF 介入 65.1% 为直接动作改写，主要集中在分解复杂问题、探索不充分等高频技能，内部化更多发生在行为纠正类 PF。  

**一句话价值**  
“将 Agent 技能转化为确定性、可审查的状态–动作干预函数，并配套多维质量信号与严格过滤机制，让经验真正‘掌控’行为，而不仅仅是‘建议’。”
