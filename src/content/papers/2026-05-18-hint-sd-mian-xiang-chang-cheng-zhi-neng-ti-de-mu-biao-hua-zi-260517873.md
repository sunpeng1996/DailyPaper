**动机**
训练长程 LLM Agent 时，只有最终任务成败的稀疏奖励，中间步骤的对错难以区分。现有方法倾向提供逐步密集反馈或全轨迹后见蒸馏，但多数中间步骤本就正确，全局反馈还常被错误放置（如在开头而非真正出错的步骤前），导致噪声更新、训练低效。因此，**何处施加监督**成为一个被忽视的关键瓶颈。

**方法**
- **后见分析 + 目标选择**：对失败轨迹，用当前策略作为后见分析器，一次性扫描全轨迹，输出一组失败相关的关键步骤以及针对每步的简短文本纠正反馈（“为何错、该怎么改”）。
- **目标化自蒸馏**：对每个选中的失败步骤，将反馈作为特权信息追加到教师模型的历史上下文，得到局部更优的 token 分布；学生模型只观察原始历史。仅在这些步骤的动作 token 上计算反向 KL 散度，不更新其他无关部分。
- **训练细节**：使用自身 EMA 策略作为反馈生成器（可扩展至更强模型），一次选择最多 3 个失败步骤。

**关键结果**
- 在长程工具调用基准 BFCL v3 和 AppWorld 上，HINT-SD-Multi 相比最强的密集逐步反馈基线，Avg@4 分别提升 32.7%（25.94→41.88）和 89.5%（9.74→18.46），Best@4 提升显著。
- 训练效率大幅改善：每训练步时间减少 2.26 倍，峰值 GPU 内存降低 1.48 倍。
- 分析表明反馈目标分布于轨迹各阶段（而非仅开头），且将反馈放在错误动作前比放在轨迹开始处能使成功率提升更高（BFCL 上 +5.99 百分点）。

**核心洞察**
“决定在何处施加反馈，是长程 Agent 后训练的一个中心设计选择。”
