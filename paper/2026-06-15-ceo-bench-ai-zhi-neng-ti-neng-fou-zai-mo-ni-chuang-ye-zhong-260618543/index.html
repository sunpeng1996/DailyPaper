    CEO-Bench: Can Agents Play the Long Game?

    CEO-Bench：AI 智能体能否在模拟创业中玩转长线经营？

    🧭 智能体长期决策评估

    Princeton University

    Haozhe Chen, Karthik Narasimhan, Zhuang Liu

    构建 500 天初创公司经营模拟，考察智能体长期规划、信息获取与自适应能力，多数模型破产，顶尖模型勉强盈利。

    Agent BenchmarkLong-horizonStartup SimulationStrategic PlanningLLM AgentPartially Observable

    📌 工作参考价值

    • 长期决策能力测试范式：在电商/推荐系统中，可参照该基准设计类似“多周期经营模拟”来评估 Agent 在面对延迟反馈、非平稳环境时的策略稳定性，例如动态定价、广告投放的长时间 A/B 测试模拟。
    • 细粒度工具与数据接口设计：该基准的 Python API 和 19 表数据库为 Agent 提供了 SQL 分析 + 精细化动作空间，可以借鉴到推荐系统 Agent 中，让模型自主查询用户行为表、调整推荐策略参数，实现闭环优化。
    • 隐藏信息推断与条件规划：模拟中智能体通过挖掘交易历史、社会媒体帖子推断市场偏好，并编写 IF-THEN 场景模拟代码；推荐系统 Agent 可类似地分析用户隐式反馈、预测不同策略的长期收益，提升自适应能力。
    • 多模型行为模式对比：研究发现顶尖模型（Opus 4.8, GPT-5.5）将 90% 开发预算用于精准客户群定向，而弱模型则“一刀切”；在广告/推荐中，可借鉴该分析维度，监控 Agent 的策略粒度与资源分配合理性。

    动机
    语言模型智能体在短周期、明确反馈的任务上已趋成熟，但真实世界中企业级任务需要：在长期不确定中规划、从噪声信号中获取信息、适应环境变化、协调多部分决策。现有基准（SWE-bench、WebArena、τ-bench）局限于短期且反馈迅速的场景，缺乏对可持续战略能力的评估。CEO-Bench 旨在填补这一空缺，通过 500 天初创公司经营模拟，一次性考察上述综合能力。

    方法

    • 环境：模拟一家订阅制软件公司，从零客户和 $1M 现金起步，周期 500 天，破产即终止。
    • 动作接口：提供 34 个 Python 工具（定价、广告投放、研发、基础设施、企业谈判、社交发布等）和 19 张业务数据库表的 SQL 查询能力；动作可细粒度到客户群甚至单个客户。
    • 世界机制：26 个异质客户群，个体客户有独立价格-质量接受曲线；隐藏信息包括渠道效率、客户心理价位、对手动态等，智能体只能通过数据分析和社交帖子推断；反馈延迟且非平稳（竞争对手冲击、需求波动、宏观经济周期）。
    • 评估模型：测试了 Claude Opus 4.8、GPT-5.5、Kimi K2.6、DeepSeek-V4 Pro 等 10 个前沿模型，每模型 3 次运行（种子固定）。使用最小化终端代理框架，每周重置上下文并保持可编辑记忆文件。

    关键结果

    • 仅 Claude Opus 4.8（最优现金 $27.8M）和 GPT-5.5（$21.3M）在最佳运行中实现盈利，且超过启发式规则基线（$15.8M），其余模型均破产或低于初始资金。
    • 性能上限估计约 $2.2B，模型仍有巨大提升空间。
    • 能力分析：高盈利模型广告渠道分配效率（43% vs 随机 20%）、未来 4 周现金预测误差仅 8%，快速检测竞争变化（1 周内），且备忘录中“if”条件语句使用频率是其他模型的 3 倍以上。
    • 行为差异：GPT-5.5 维持稳定客户群，Opus 4.8 中途客户归零后切换“收割”模式仍获利；弱模型（如 Opus 4.7）则长期固守保守策略。

    核心结论：当前智能体缺乏将短期工具能力组合为长期连贯策略的能力，CEO-Bench 为评估智能体“驱动系统持续演进”的智力水平提供了第一步。
